# 디자인 가이드라인: 시니어 친화 + 모던(촌스럽지 않게)

> 작성자 관점: `personas/designer.md` (Maya Designer)  
> 목표: **어르신도 쉽게**, **청년에게도 세련되게** 보이는 “신뢰/안전” 중심 UI

## 0) 핵심 디자인 원칙
- **Clarity over cleverness**: 장식보다 명확성(특히 텍스트/버튼)
- **Progressive disclosure**: 필요한 것만 먼저, 상세는 펼치기(요약 + 원문)
- **One primary action per screen**: 한 화면에 핵심 CTA 1개
- **Confidence UI**: 불확실은 숨기지 말고 “확인 필요”로 투명하게 표시
- **Trust cues**: 무저장 OCR, TTL 10분, 환자만 재발급 같은 정책을 UI에서 “짧게” 상시 노출

## 1) 정보 구조(IA) 권장
### 1.1 환자 플로우(우선순위 B → C → A)
- **B(첫 사용)**: 조제내역서 촬영 → 내 약력 생성
- **C(지속)**: 복약 알림 + 경과 기록
- **A(피크)**: 진료실에서 “의사에게 보여주기(QR)”로 공유

### 1.2 핵심 화면(최소 세트)
1) 온보딩/촬영 가이드(조제내역서 1순위)
2) 업로드/분석 진행 화면(큰 진행바, 재시도)
3) **첫 결과 화면(결정: A+C)**
   - 현재 복용중 요약(큰 카드)
   - 복약 알림 설정(원탭)
4) 대기실 설문(2~3분)
5) 의사 공유(QR 생성/만료)
6) 기록(90일 타임라인) + “확인 필요” 리스트

## 2) 시니어 친화 UI 룰(접근성)
- **폰트**
  - 기본 17–18pt, 핵심 헤딩 22–24pt
  - 줄간격 1.35–1.5
- **터치 타깃**
  - 최소 44px(권장 48px)
  - 버튼 간 간격 확보(오클릭 방지)
- **색 대비**
  - 텍스트 대비 강하게(최소 WCAG AA 권장)
  - “회색 글씨”로 중요한 정보 숨기지 않기
- **동작 피드백**
  - 업로드/분석은 1초 이상이면 반드시 로딩 상태/남은 단계 표시
  - 실패 시 “왜 실패했는지 + 어떻게 해결하는지” 한 문장 + 재시도 버튼
- **언어**
  - 의료 용어 최소화, 단어를 짧게
  - 예: “OCR” 대신 “사진 분석”

## 3) 모던하게 보이기(촌스럽지 않게)
- **색감**
  - “병원 느낌”의 과도한 파란색/초록색 남발 금지
  - 권장 팔레트: **뉴트럴(오프화이트/차콜) + 1개의 포인트 컬러**(블루그레이/인디고 계열)
- **타이포/레이아웃**
  - 카드 기반, 여백을 넉넉히(숨 쉴 공간)
  - 그림자 과다 사용 금지(플랫+소프트)
- **아이콘**
  - 라인 아이콘 통일, 의미가 분명한 것만
- **마이크로카피**
  - 과장/마케팅 문구 대신 “신뢰/안전/통제”를 강조
  - 예: “사진은 저장하지 않아요(분석 후 즉시 삭제)”

## 4) 상태/배지 시스템(신뢰도 표시)
- **확인 필요(verify)**: 노랑/앰버 계열(주의), 텍스트와 함께 사용
- **완료/정상**: 그린 사용은 최소, 체크 아이콘 중심
- **만료/시간 제한**: 레드 남발 금지, “시간이 지나면 자동으로 닫혀요”로 표현

## 5) 첫 결과 화면(A+C) 디자인 스펙(핵심)
### 5.1 상단(즉시 가치)
- 큰 타이틀: “현재 복용중”
- 큰 카드(1~3개 약만 우선 노출, 나머지는 “더보기”)
- 각 약 카드: 약 이름(원문), 기간/횟수(가능할 때), “확인 필요” 배지(있을 때)

### 5.2 바로 아래(지속 사용)
- Primary CTA: “복약 알림 켜기”
  - 원탭으로 기본 시간 추천(아침/점심/저녁)
  - “나중에”는 텍스트 링크로 작게

### 5.3 신뢰/프라이버시(짧게 고정)
- 작은 한 줄: “사진은 분석 후 저장하지 않아요”

## 6) 대기실 설문 UX(2~3분)
- 한 화면에 질문 1개(또는 2개) 원칙
- 선택형(라디오/칩) + 필요 시 짧은 서술
- 진행률(예: 3/7) 반드시 표시
- 스킵 가능한 항목은 “선택” 표시

## 7) 의사 공유(QR) UX
- 큰 QR + 만료 카운트다운(10분)
- 안내 문구(짧게): “진료실에서 의사/간호사에게 보여주세요”
- 재발급: 환자만 가능(설명 포함: “재발급하면 이전 코드는 바로 꺼져요”)

## 8) 의료진 화면(요약 + 원문 펼치기) 톤
- 요약은 1장에 끝(정보 과다 금지)
- “환자 작성” 라벨을 모든 서술 텍스트에 명확히
- “확인 필요”는 상단에 모아서 보여주고 클릭 시 해당 위치로 스크롤

## 9) 컴포넌트 가이드(요약)
- 버튼: Primary 1개, Secondary는 아웃라인
- 카드: 제목/본문/배지 일관된 패턴
- 배지: 텍스트+아이콘, 색만으로 의미 전달 금지

## 10) 대회 제출용 디자인 메시지(한 문장)
- “어르신도 대기실에서 2분이면 끝—하지만 화면은 모던하고, ‘신뢰’가 느껴지게.”
