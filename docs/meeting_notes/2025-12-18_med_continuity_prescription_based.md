# 회의록: 처방전/조제내역서 기반 연속진료(MVP) (2025-12-18)

## 참석자

- **CEO(Elon-style)**: 비전/제품 방향, 리스크 감수/속도 우선
- **PM(Athena)**: 규제/실행/스코프 조율
- **사용자(당신)**: 제품 방향 결정, MVP 제약/운영 선택

## 문제 정의

- **지방 의료 접근성 문제**: 지역에 병원이 부족하고 의료 이동 비용이 커서, A병원에서 받은 진료 맥락이 B병원에서 끊김
- **목표**: A에서의 진료 맥락을 B에서 이어 받을 수 있게 하여 **중복검사/진료 지연을 줄이고 연속진료를 개선**

## 핵심 제약/전제

- EMR 직접 연동은 복잡도/온보딩/규제·계약 이슈로 **MVP에서 제외**
- 응급 상황(의식불명 등)은 **MVP 범위 제외**
- 사용자(환자)가 **직접 앱 사용**(보호자/대리 기본 시나리오는 우선순위 낮음)

## 데이터 접근 전략(합의)

- 환자가 보유한 문서 기반:
  - **종이 처방전 사진**
  - **약국 조제내역서(복약안내문/조제내역서/영수증 등) 사진**
- 온보딩/기본 촬영 우선순위(확정): **조제내역서(약국 종이) → 처방전**
- 환자가 “의사가 뭐라고 했는지”를 **간단 메모로 추가**
- 시스템은 위 자료를 기반으로:
  - **약력(언제/무슨 약/얼마나) 타임라인**
  - **환자 경과 추적(복용 후 호전/악화 시점 등)**
  - **B병원 의사가 참고할 1장 요약 화면** 제공

## AI 사용 범위 논의 및 결론

- 사용자 제안: 처방전의 약을 기반으로 **AI가 진단/상태를 유추**하여 참고 제공(C안)
- PM 리스크 지적:
  - “질환 가능성 추정”은 환자에게 숨기고 **의사용 참고로만 제공해도** 제품의 의도된 사용 목적이 진단/치료 결정지원에 걸리면
    **소프트웨어 의료기기(SaMD)·AI 의료기기 규제 트랙(MFDS)로 들어갈 가능성**이 있음
- 최종 결론(사용자 결정): **1번(허가 트랙) = NO**
  - MVP에서는 **질환/진단 후보 제시 금지**
  - 대신 “질문을 통해 환자 데이터를 수집 → 구조화/요약 → 의사에게 제공” 방식으로 간다

## 질문/설문 전략(합의)

- 질문은 **환자에게** 하고, 그 응답을 **의사에게** 보여준다 (맞음)
- 환자에게 보여주는 출력은 최소화; 의사 참고용을 중심으로 설계
- 설문 타이밍: **병원 대기 중**
- 표시 방식: **요약 + 원문 펼치기**

## 병원/의사 열람 방식(합의)

- 환자 흐름: **앱에서 병원 선택 후** 설문 시작
- 의사 열람(MVP): **로그인 제외**
- “MVP 쉬운 방식” 선택:
  - 환자가 병원에서 **QR/코드**를 보여주고 의료진이 스캔하여 열람(자동 전송/병원 시스템 연동은 MVP에서 제외)

## 원본 이미지 저장 정책(합의)

- **A안 선택**: 원본 이미지 서버 저장 **하지 않음**
  - 구현 결정(확정):
    - **서버 업로드 후 즉시 OCR 처리 → 원본 이미지 즉시 삭제(무저장)**, 추출 텍스트만 저장

## 의사 화면 필요(합의)

- 의사 전용 “1장 요약 화면” **필수**
- 구성은 PRD/스펙 문서에서 필드 고정으로 확정 예정

## 미해결/추가 결정 필요(다음 회의 안건)

- (해결) OCR 처리 방식 선택 완료: **서버 즉시 처리(무저장)**
- (해결) 약품명 정규화 수준(MVP): **OCR 원문 그대로(A안)**
- (해결) 의료진 열람용 링크/코드 보안 가드레일
  - TTL 기본값: **10분(확정)**
  - 1회성 규칙: **TTL(10분) 내 재열람 허용**, TTL 만료 시 열람 불가
  - 재발급 규칙: **환자만 재발급 가능**, 재발급 즉시 이전 토큰 폐기
- (남음) 설문 문항/분기(2~3분) 최적화 및 필수/선택 확정
- (남음) “의사 메모(환자 작성)” 표현 가이드(출처 라벨링, 오인 방지 문구) 확정

## B2C 차별화/성장 논의(추가 합의)

- 초기 우려: “연속진료/병원 이동”은 강한 가치지만 **저빈도 이벤트**라 B2C에서 처음 설치 유인이 약할 수 있음
- 합의한 B2C 우선순위: **B → C → A**
  - **B(Activation)**: “내 약 정리(약력 자동 생성)”
  - **C(Retention)**: “복약 후 경과 추적/재진 대비”
  - **A(Peak Moment)**: “진료실에서 의사에게 1장 요약 공유”

## 온보딩/첫 사용 훅(추가 합의)

- 온보딩 기본 촬영: **조제내역서(약국 종이) 1순위** (처방전은 선택/추가)
- 첫 결과 화면(환자): **A + C 결합**으로 “즉시 체감 가치” 제공
  - **A**: “현재 복용중” 약을 크게 요약(가능한 범위에서 기간/횟수/남은 일수 등)
  - **C**: “복약 알림 설정”을 즉시 제공(원탭 설정, 기본값 추천)
- 의도: “병원 이동이 없어도 오늘부터 쓸 이유”를 만들고, 이후 “의사에게 보여주기”로 자연스럽게 연결

## Phase 1(다음 스텝) 기능 방향 합의: OTC 복약편의 보조

- 전제: **일반의약품(OTC) 한정**, **효능군 유사**, 목표는 **복약 편의**
- 설계(확정):
  - 입력: **처방약 효능군 기반**
  - 출력: **브랜드/제품명 추천은 하지 않음**
    - 효능군/카테고리 수준의 안내 + “약사에게 확인할 질문/주의 체크리스트”
  - 가드레일: “대체 가능/동일 효과” 등 단정 표현 금지, 중복/상호작용은 “확인 필요”로만 표시

## PRD에 추가된 MVP 확장 기능(요약)

- 11.1 환자 인증: Replit Auth(구글/깃허브/애플), 세션(PostgreSQL), 비로그인 랜딩 분리
- 11.2 빠른 처방 기록(Quick Prescription Recording): 접수 없이 사진 업로드→OCR 검토→메타(병원/약국/증상/진단) 입력, OCR 미리보기 API
- 11.3 증상별 과거 기록 조회: `chiefComplaint` 기반 조회 + 약물 통계
- 11.4 알림 시스템: 복약/재진/조회 알림, 설정, 읽음 처리
- 11.5 진단명 필드: `doctorDiagnosis` 추가
- 11.6 OCR 신뢰도 강화: 80% 미만 “검증 필요” 뱃지 + 분포 시각화
